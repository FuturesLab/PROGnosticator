# TinyGo Translation Campaign

Dependencies:
- **Go**: [install from here](https://go.dev/doc/install).
- **TinyGo**: [install from here](https://github.com/tinygo-org/tinygo).


This script automates the process of: Compiling Go programs using the standard Go compiler -> Translating them into WebAssembly using TinyGo -> Running both the native Go and TinyGo-generated WASM binaries -> Comparing outputs to detect divergences

---

## 🔧 How to Run

```bash
python3 PROGnosticatoring.py <input_Go_folder> <campaign_id>
```

- `<input_Go_folder>`: Directory containing PROGnosticator-generated `.go` files.
- `<campaign_id>`: A unique identifier to organize results (e.g., `1` etc.)

We here provide some sample PROGnosticator-generated Go programs in `prog_sample_programs` folder.


- You must pass two arguments when running the script:

```bash
  python3 PROGnosticatoring.py <input_Go_folder> <campaign_id>

  # for example
  python3 prog_sample_programs 1
```

---

## 📁 Output Directory Structure

The script creates a directory:

```
campaign_<campaign_id>/
```

### `processing/temp_<filename>/`
Temporary directory for each Go file, containing:
- `runner.go`: The original Go source
- `runner_go`: Compiled native Go binary
- `runner.wasm`: WASM binary generated by TinyGo

### `tinygo_crashes/`
(Not used in this script but may be created in extensions.)

### `tinygo_divergence/`
(Not used in this script but may be created in extensions.)

---

## 📄 Log Files

- `log_go_build_failure.txt`  
  → Go files that failed to build with `go build`.

- `log_tinygo_build_failure.txt`  
  → Files that failed to build with TinyGo to WASM.

- `log_bad_Go_exe.txt`  
  → Go binaries that failed to run or timed out.

- `log_wasm_crash.txt`  
  → WASM binaries that failed to run using `wasmtime`.

- `log_diverge_output.txt`  
  → Output mismatches between Go and WASM execution.

- `summary.txt`  
  → Summary of total files processed and the result of each phase.
